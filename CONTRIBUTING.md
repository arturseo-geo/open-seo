# Contributing to open-seo

Thank you for your interest in contributing! This project is maintained by [Artur Ferreira](https://github.com/arturseo-geo) at [The GEO Lab](https://thegeolab.net).

## How to Contribute

### Reporting Issues
- Use the [issue tracker](https://github.com/arturseo-geo/open-seo/issues) to report bugs or suggest features
- Check existing issues before creating a new one
- Include steps to reproduce for bug reports

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes
5. Commit with a clear message (`git commit -m "feat: add your feature"`)
6. Push to your fork (`git push origin feature/your-feature`)
7. Open a Pull Request

### Commit Message Convention
- `feat:` — new feature
- `fix:` — bug fix
- `docs:` — documentation only
- `refactor:` — code change that neither fixes a bug nor adds a feature
- `chore:` — maintenance tasks

### Code Style
- Follow existing patterns in the codebase
- Keep changes focused — one PR per feature/fix

## Questions?

Open an issue or reach out at artur@thegeolab.net.
